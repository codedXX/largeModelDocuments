## 2. 注释（Comments）

### 2.1 什么是注释？
**注释**是写在程序中对代码进行解释说明的文字，方便自己和其他人查看，以便理解程序。

### 2.2 注释的分类与写法
Java 中有三种注释写法：
1. **单行注释**：`// 注释内容`（只能写一行）
2. **多行注释**：`/* 注释内容 */`（可以包含多行）
3. **文档注释**：`/** 注释内容 */`（通常写在类、方法、成员变量上方）
   - **特殊作用**：文档注释的内容可以被 `javadoc` 工具提取到程序说明文档中去。

### 2.3 注释的特点
- **不影响程序的执行**：注释只存在于编写代码阶段。`javac` 编译后的 `.class` 字节码文件中**已经没有注释了**，因此注释完全不会影响程序的执行速度或逻辑。

**代码示例：**
```java
/**
 * 这是一个文档注释
 * 目标：让同学们学会使用注释
 */
public class CommentDemo {
    public static void main(String[] args) {
        // 这是一行单行注释：往下控制台打印一下我的心情
        System.out.println("开始正式学Java程序，很哈皮!");
        
        /*
           这是一个多行注释：
           窗前明月光
           疑是地上霜
        */
        System.out.println("HelloWorld");
    }
}
```

## 3. 字面量（Literals）

### 3.1 什么是字面量？
**字面量**就是程序中能**直接书写的数据**。学习字面量主要是搞清楚 Java 程序中各类数据的书写格式。

### 3.2 常见字面量分类及书写格式

| 数据类型 | 生活中/说明 | 程序中书写示例 | 格式要求 |
| --- | --- | --- | --- |
| **整数** | 666, -88 | `666` 或 `-88` | 写法一致 |
| **小数** | 13.14, -5.21 | `13.14` 或 `-5.21` | 写法一致 |
| **字符** | A, 0, 我 | `'A'`、`'0'`、`'我'` | **必须使用单引号**，且有且仅能有一个字符。 |
| **字符串** | HelloWorld | `"HelloWorld"` | **必须使用双引号**，内容可有可无。 |
| **布尔值** | 真、假 | `true` 或 `false` | 只有两个值：`true`（真）和 `false`（假）。 |
| **空值** | 无 | `null` | 代表一个特殊的空值。 |
| **特殊字符**| 换行、制表符 | `\n` (换行)、`\t` (制表) | 属于转义字符。 |

## 4. 变量详解（Variables）

### 4.1 什么是变量？
**变量**就是内存中的一块区域，可以理解成一个**盒子**，用来装程序要处理的数据的。

### 4.2 变量的定义格式
```java
// 数据类型 变量名称 = 数据;
int age = 18;
```

### 4.3 为什么使用变量？
使用变量记要处理的数据，可以使**编写的代码更灵活，管理代码更方便**。
- **核心特点**：变量里装的数据是**可以被替换**（修改）的。

**代码示例：**
```java
int age = 18;
age = 19;             // 变量的值被修改为 19
System.out.println(age);
age = age + 1;        // 变成了 20
System.out.println(age);
```

### 4.4 使用变量的注意事项（重点）
1. 变量**要先声明才能使用**。
2. 变量是什么类型，就必须装什么类型的数据（不能把小数放进整数变量中）。
3. 变量的有效范围是从定义开始，到其所在的 `}` 截止。
4. 在同一个有效范围内，**变量名不能重复**。
5. 变量定义时可以不赋初始值，但在**使用时，变量里必须有值**。

## 5. 计算机底层的存储原理

### 5.1 二进制与单位
- 计算机底层数据均采用**二进制**存储（使用 0 和 1，逢 2 进 1）。
- **十进制转二进制**算法：**除二取余法**。
- **存储单元**：
  - **比特位**（bit / b）：最小的 0 或 1。
  - **字节**（Byte / B）：计算机中表示数据的**最小单元**。
  - **换算关系**：`1B = 8b`，`1KB = 1024B`，`1MB = 1024KB`，以此类推。

### 5.2 字符、图片、音频如何存储？
- **字符**：存储的是 **ASCII 编码表**中对应数字的二进制形式。
  - `'A'` 对应 `65`
  - `'a'` 对应 `97`
  - `'0'` 对应 `48`
- **图片和音视频**：也是将像素颜色（如 RGB 0~255）或声波数据转化为二进制进行存储。

### 5.3 进制拓展
- **八进制**：每 3 位二进制作为一个单元（0-7）。Java 中以 `0` 开头。
- **十六进制**：每 4 位二进制作为一个单元（0-9, A-F）。Java 中以 `0X` 或 `0x` 开头。
- **二进制表示**：Java 中以 `0B` 或 `0b` 开头。

## 7. 关键字与标识符

### 7.1 关键字（Keywords）
- **概念**：Java 语言自己保留使用的一些词汇，具有**特殊作用**（如 `public`、`class`、`int` 等）。
- **注意**：关键字已经被 Java 占用，**不能用来作为自定义的类名、变量名**，否则会报错！

### 7.2 标识符（Identifiers）
- **概念**：标志符就是程序中取的名字（如类名、变量名等）。
- **命名规则**（必须遵守）：
  1. 一般由数字、字母、下划线（`_`）和美元符（`$`）组成。
  2. **不能以数字开头**。
  3. 不能是关键字。
  4. 不能包含特殊字符（如 `%`、`&` 等）。
- **命名规范**（建议遵守）：
  - **变量名**：首字母小写，遵循“**小驼峰模式**”（例如：`studyNumber`）。
  - **类名**：首字母大写，遵循“**大驼峰模式**”（例如：`HelloWorld`）。
