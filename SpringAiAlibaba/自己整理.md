# 01ChatModel

## 问 1：`dashScopeChatModel` 是怎么注入 api-key 的？由哪个依赖完成？

> 01ChatModel 子模块里：
>
> ```java
> @Resource //阿里云百炼
> private ChatModel dashScopeChatModel;
> ```
>
> 这里是怎么注入 api-key 的，又是哪个依赖在做注入？

**🔹 一句话结论**

yaml 里的 `spring.ai.dashscope.api-key` → 绑定到 `DashScopeConnectionProperties.apiKey` → `DashScopeChatAutoConfiguration` 用它 build 出名为 `dashScopeChatModel` 的 Bean → `@Resource` 按字段名注入到 Controller。整个过程由随 starter 引入的 `spring-ai-alibaba-autoconfigure-dashscope` 完成，你一行配置类都不用写。

**🔹 哪个依赖在做注入**

是 pom 里的 **`spring-ai-alibaba-starter-dashscope`**。它本身是个"空壳" starter，真正干活的是它传递引入的两个 jar：

| jar | 作用 |
| --- | --- |
| `spring-ai-alibaba-dashscope` | 提供 `DashScopeChatModel`、`DashScopeApi` 等实现类 |
| `spring-ai-alibaba-autoconfigure-dashscope` | 提供**自动配置类**，Bean 由它注册 |

**🔹 api-key 一路注入进去的完整链路**

**① Spring Boot 启动时加载自动配置类**

`@SpringBootApplication` 开启自动配置，扫到 autoconfigure jar 里 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports` 登记的 `DashScopeChatAutoConfiguration`。

**② 配置属性绑定 —— yaml 里的 api-key 被读进 Java 对象**

```java
@ConfigurationProperties("spring.ai.dashscope")           // ← 前缀
public class DashScopeConnectionProperties extends DashScopeParentProperties {
    // 父类里有 private String apiKey;
}
```

yaml 里写的 `spring.ai.dashscope.api-key: sk-xxx`，按"前缀 + 字段名"（`api-key` → `apiKey`）自动绑定到这个属性类。

**⭐③ 自动配置类用属性创建 Bean**

```java
@ConditionalOnMissingBean                    // 你自己没定义同类 Bean 时才生效
public DashScopeChatModel dashScopeChatModel(...) {
    var dashscopeModel = DashScopeChatModel.builder()...  // 内部用 apiKey 构建 DashScopeApi
}
```

⭐方法名就是 Bean 名：**`dashScopeChatModel`**。api-key 在这一步被塞进 `DashScopeApi`，之后每次调用都会放到 HTTP 请求头 `Authorization: Bearer sk-xxx` 里发给百炼。

**④ Controller 注入**

```java
@Resource
private ChatModel dashScopeChatModel;   // 字段名 == Bean 名，精确命中
```

`@Resource` 先按**字段名**找 Bean，`dashScopeChatModel` 正好对上第 ③ 步注册的名字，注入成功。

> 对比：ToolCalling12 那边字段名叫 `chatModel`，按名字找不到，退化成按类型找，撞上 3 个候选才报错。

**⭐注意：**

* DashScopeChatModel类是实现了ChatModel 接口的。



# 总结

## @Resource和@Autowired的区别是什么？各自是怎么找bean的?

| `@Autowired`     | `@Resource`                   |                                           |
| :--------------- | :---------------------------- | ----------------------------------------- |
| 来源             | Spring 自带                   | JDK / JSR-250（`jakarta.annotation`）     |
| **默认查找方式** | **先按类型**（byType）        | **先按名字**（byName）                    |
| 找不到时的兜底   | 类型匹配到多个 → 再按名字筛   | 名字找不到 → 再按类型找                   |
| 指定名字的搭档   | 配合 `@Qualifier("beanName")` | 自带 `name` 属性：`@Resource(name="xxx")` |
| 允许找不到       | `@Autowired(required=false)`  | 无此选项                                  |

**⭐注意：**

* `@Autowired`的按类型去找，意思是按类型去匹配，并不是说用类型去对应名字
  * 比如说：@Autowired private ChatModel dashScopeChatModel;这里一开始是**去 IOC 里找类型是 `ChatModel` 的 Bean，这一步不带任何名字**。





## Spring 注册 Bean 时的命名规则

| 写法                            | Bean 名称            | 说明                                          |
| :------------------------------ | :------------------- | :-------------------------------------------- |
| `@Bean` + `public X deepSeek()` | `deepSeek`           | **默认 = 方法名,原样保留大小写**,不做任何转换 |
| `@Bean(name = "deepseek")`      | `deepseek`           | 显式指定时**以注解为准**,忽略方法名           |
| `@Bean("deepseek")`             | `deepseek`           | `name =` 可省略,效果相同                      |
| `@Bean(name = {"a", "b"})`      | `a`(主名)+ `b`(别名) | 可起多个名字                                  |



## Spring 注册 Bean 时如何使用@Qualifier注解

```java
@Configuration
public class SaaLLMConfig {
    // 模型名称常量定义
    private final String DEEPSEEK_MODEL = "deepseek-v4-flash";
    private final String QWEN_MODEL = "qwen3-max";
    @Value("${spring.ai.dashscope.api-key}")
    private String apiKey;

    @Bean(name = "deepseek")
    public ChatModel deepSeek() {
        return DashScopeChatModel.builder()
                .dashScopeApi(DashScopeApi.builder()
                        .apiKey(apiKey)
                        .build())
                .defaultOptions(
                        DashScopeChatOptions.builder().withModel(DEEPSEEK_MODEL).build()
                )
                .build();
    }

    @Bean(name = "qwen")
    public ChatModel qwen() {
        return DashScopeChatModel.builder().dashScopeApi(DashScopeApi.builder()
                        .apiKey(apiKey)
                        .build())
                .defaultOptions(
                        DashScopeChatOptions.builder()
                                .withModel(QWEN_MODEL)
                                .build()
                )
                .build();
    }

    /**
     @Qualifier("deepseek") 是什么意思?
     它的作用是:当按类型注入会遇到多个候选 Bean 时,用名字精确指定要注入哪一个。
     */
    @Bean(name = "deepseekChatClient")
    public ChatClient deepseekChatClient(@Qualifier("deepseek") ChatModel deepSeek) {
        return ChatClient.builder(deepSeek)
                .defaultOptions(ChatOptions.builder()
                        .model(DEEPSEEK_MODEL)
                        .build())
                .build();
    }
}
```

**🔹这里的@Qualifier("deepseek")是什么意思，为什么要这样写?**



它的作用是:**当按类型注入会遇到多个候选 Bean 时,用名字精确指定要注入哪一个。**

为什么这里必须写

看这个方法的参数:

```java
public ChatClient deepseekChatClient(@Qualifier("deepseek") ChatModel deepSeek) {
//                                                          ^^^^^^^^^ 类型
```

Spring 要给这个 `ChatModel deepSeek` 参数注入一个 Bean。但你的容器里 **`ChatModel` 类型有 3 个 Bean**:

| Bean 名              | 来源                                          |
| :------------------- | :-------------------------------------------- |
| `deepseek`           | 你 SaaLLMConfig 里的 `@Bean(name="deepseek")` |
| `qwen`               | 你 SaaLLMConfig 里的 `@Bean(name="qwen")`     |
| `dashScopeChatModel` | starter 自动配置                              |

Spring 默认**按类型**找 Bean,一找发现 3 个都是 `ChatModel`,**不知道该用哪个** → 就会报你之前那个错:`required a single bean, but 3 were found`。

`@Qualifier("deepseek")` 就是告诉 Spring:**"这 3 个里,给我注入名字叫 `deepseek` 的那个"**。歧义就消除了。
